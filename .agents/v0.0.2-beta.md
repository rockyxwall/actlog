# ACTLog v0.0.2-beta — Installer, Portable Mode, No Terminal

## Goals

1. **No terminal window** in release builds (Windows subsystem, conditional)
2. **Proper installer** via Inno Setup (no admin required, auto-start, uninstall)
3. **Explicit portable mode** (data stays next to exe when `portable.flag` present)
4. **Structured logging** with `log` + `simplelog` (console in debug, file in release)
5. **Installed mode** stores data in `%APPDATA%/actlog/` (lowercase, per AGENTS.md)

---

## Changes

### 1. Hide Console Window (conditional)

**File**: `src/main.rs`

Add at top:
```rust
#![cfg_attr(not(debug_assertions), windows_subsystem = "windows")]
```

- `cargo build` (debug) → terminal visible for development
- `cargo build --release` → no terminal

---

### 2. Structured Logging

**Files**: `Cargo.toml`, `src/logging.rs` (new), `src/main.rs`, `src/server.rs`, `src/tracker.rs`

- Add `log` and `simplelog` crates to `Cargo.toml`
- Create `src/logging.rs` with `init_logging()` that:
  - Gets data dir from `db::get_data_dir()`
  - Initializes `simplelog` with `WriteMode::Write` (truncate on each startup — prevents unbounded growth)
  - **Portable**: writes `actlog.log` next to exe
  - **Installed**: writes `%APPDATA%/actlog/actlog.log`
- Call `logging::init_logging()` at start of `main()`
- Replace all `println!`/`eprintln!`:
  - `info!(...)` for status messages
  - `error!(...)` for errors
  - `warn!(...)` for warnings

---

### 3. Portable vs Installed Mode

**File**: `src/db.rs`

Rename `get_app_dir()` → `get_data_dir()`:

```rust
pub fn get_data_dir() -> Result<PathBuf> {
    let exe_dir = current_exe().parent()...;
    let flag = exe_dir.join("portable.flag");
    if flag.exists() {
        Ok(exe_dir)  // portable: data next to exe
    } else {
        let appdata = env::var("APPDATA")?;
        let dir = PathBuf::from(appdata).join("actlog");  // lowercase
        fs::create_dir_all(&dir)?;
        Ok(dir)  // installed: %APPDATA%/actlog/
    }
}
```

Update all callers: `init_db()`, `open_reader_conn()`, `get_or_create_device_id()`

---

### 4. Inno Setup Installer

**New file**: `installer/actlog.iss`

```
PrivilegesRequired=lowest          ← no admin prompt
Install to: {localappdata}\Programs\actlog\
Data dir:   %APPDATA%/actlog/      ← no portable.flag present
Auto-start: Create .lnk in %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\
Uninstall:  Control Panel entry
```

Key decisions:
- **No admin required** (user-local install)
- **Startup via .lnk** (not registry — per AGENTS.md)
- **Installer does NOT include** `portable.flag`

---

### 5. Portable Release Package

- Zip containing `actlog.exe` + empty `portable.flag`
- User unzips anywhere, data stays local
- No registry writes, no startup entry

---

### 6. Cargo.toml Updates

- Keep version as-is (release tag `v0.0.2-beta` is separate)
- Add `log`, `simplelog` to `[dependencies]`

## Build Commands

```batch
:: Release build
cargo build --release

:: Portable package (manual)
copy target\release\actlog.exe dist\portable\
echo. > dist\portable\portable.flag
:: zip dist\portable\ → ACTLog-v0.0.2-beta-portable.zip

:: Installer (requires Inno Setup)
copy target\release\actlog.exe dist\installer\
iscc installer\actlog.iss
:: outputs ACTLog-v0.0.2-beta-installer.exe
```
